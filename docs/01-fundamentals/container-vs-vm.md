# 容器 vs 虚拟机

> 两种虚拟化技术的对比

---

## 架构对比

### 虚拟机

```
┌─────────────────────────────────────────────┐
│   App A  │   App B  │   App C               │
│  Bin/Lib │  Bin/Lib │  Bin/Lib              │
├──────────┴──────────┴──────────┬────────────┤
│         Guest OS A             │ Guest OS B │
│        (Full Kernel)           │ (Full K)   │
├────────────────────────────────┴────────────┤
│              Hypervisor                     │
│         (VMware, KVM, Xen)                  │
├─────────────────────────────────────────────┤
│                Host OS                      │
├─────────────────────────────────────────────┤
│               Hardware                      │
└─────────────────────────────────────────────┘
```

### 容器

```
┌─────────────────────────────────────────────┐
│   App A  │   App B  │   App C               │
│  Bin/Lib │  Bin/Lib │  Bin/Lib              │
├─────────────────────────────────────────────┤
│          Container Runtime                  │
│         (Docker, containerd)                │
├─────────────────────────────────────────────┤
│              Host OS Kernel                 │
│         (Shared Kernel)                     │
├─────────────────────────────────────────────┤
│               Hardware                      │
└─────────────────────────────────────────────┘
```

---

## 关键差异

| 特性 | 容器 | 虚拟机 |
|------|------|--------|
| **启动时间** | 秒级 | 分钟级 |
| **资源占用** | MB 级 | GB 级 |
| **隔离级别** | 进程级 | 硬件级 |
| **性能** | 接近原生 | 有开销 |
| **安全性** | 共享内核 | 强隔离 |
| **密度** | 高 (单节点数百) | 低 (单节点数十) |

---

## 使用场景

| 场景 | 推荐 | 原因 |
|------|------|------|
| 微服务 | 容器 | 快速启动、高密度 |
| 遗留应用 | VM | 需要完整 OS |
| 安全隔离 | VM | 强隔离 |
| CI/CD | 容器 | 快速、一致 |
| 多租户 | VM | 边界清晰 |

---

## 混合方案

```
┌─────────────────────────────────────────────┐
│              Kubernetes                      │
│  ┌───────────────────────────────────────┐  │
│  │         VM (KubeVirt)                  │  │
│  │   ┌──────────┐  ┌──────────┐          │  │
│  │   │ Container│  │ Container│          │  │
│  │   └──────────┘  └──────────┘          │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

**容器化 + 虚拟化** 结合两者优势。

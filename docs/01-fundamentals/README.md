## 什么是容器？

容器是一种**操作系统级虚拟化**技术，它允许在单一操作系统实例上运行多个隔离的用户空间环境。

### 容器 vs 虚拟机

| 特性 | 容器 | 虚拟机 |
|------|------|--------|
| 启动速度 | 秒级 | 分钟级 |
| 资源占用 | MB 级 | GB 级 |
| 隔离级别 | 进程级 | 硬件级 |
| 性能 | 接近原生 | 有虚拟化开销 |
| 密度 | 单机数百个 | 单机数十个 |

### 容器核心原理

```
┌─────────────────────────────────────────┐
│           应用 A    应用 B    应用 C      │
│           (nginx)  (redis)   (app)      │
├─────────────────────────────────────────┤
│  用户空间隔离 (Namespace)                │
│  - PID: 进程隔离                         │
│  - NET: 网络隔离                         │
│  - IPC: 进程间通信隔离                    │
│  - MNT: 文件系统挂载隔离                  │
│  - UTS: 主机名隔离                        │
│  - USER: 用户隔离                        │
├─────────────────────────────────────────┤
│  资源限制 (Cgroups)                      │
│  - CPU: 计算资源限制                      │
│  - Memory: 内存限制                       │
│  - IO: 磁盘 IO 限制                       │
│  - Network: 网络带宽限制                   │
├─────────────────────────────────────────┤
│         Linux 内核 (共享)                │
└─────────────────────────────────────────┘
```

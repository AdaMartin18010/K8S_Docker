## 2. 控制平面与数据平面分离

```
┌─────────────────────────────────────────────────────────────────────────────────────────────┐
│                           控制平面 vs 数据平面                                                │
├─────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                              │
│   ┌─────────────────────────────────────────────────────────────────────────────────────┐  │
│   │                              控制平面 (Control Plane)                               │  │
│   │                              负责: 决策、配置、状态管理                               │  │
│   │                                                                                      │  │
│   │   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐            │  │
│   │   │ K8s API     │   │ Istiod      │   │ ArgoCD      │   │ Prometheus  │            │  │
│   │   │ Server      │   │ (Istio CP)  │   │ (GitOps)    │   │ (Monitoring)│            │  │
│   │   │             │   │             │   │             │   │             │            │  │
│   │   │ • 状态管理   │   │ • 配置下发   │   │ • 同步控制   │   │ • 指标收集   │            │  │
│   │   │ • 调度决策   │   │ • 证书管理   │   │ • 应用状态   │   │ • 告警规则   │            │  │
│   │   │ • 策略执行   │   │ • 服务发现   │   │ • 回滚控制   │   │ • 查询接口   │            │  │
│   │   └─────────────┘   └─────────────┘   └─────────────┘   └─────────────┘            │  │
│   │                                                                                      │  │
│   │   特点: 高可用要求、资源敏感、变更不频繁、大脑角色                                     │  │
│   └─────────────────────────────────────────────────────────────────────────────────────┘  │
│                                           │                                                  │
│                                           │ 配置/策略下发                                     │
│                                           ▼                                                  │
│   ┌─────────────────────────────────────────────────────────────────────────────────────┐  │
│   │                              数据平面 (Data Plane)                                  │  │
│   │                              负责: 流量处理、业务逻辑                                 │  │
│   │                                                                                      │  │
│   │   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐            │  │
│   │   │ kube-proxy  │   │ Envoy       │   │ App Pod     │   │ Node        │            │  │
│   │   │ (Service    │   │ (Sidecar/   │   │ (业务容器)   │   │ Exporter    │            │  │
│   │   │  Proxy)     │   │  ztunnel)   │   │             │   │             │            │  │
│   │   │             │   │             │   │             │   │             │            │  │
│   │   │ • 流量转发   │   │ • 负载均衡   │   │ • 业务处理   │   │ • 指标暴露   │            │  │
│   │   │ • 连接管理   │   │ • mTLS加密   │   │ • 数据处理   │   │ • 健康检查   │            │  │
│   │   │ • 会话保持   │   │ • 熔断限流   │   │ • 请求响应   │   │ • 日志输出   │            │  │
│   │   └─────────────┘   └─────────────┘   └─────────────┘   └─────────────┘            │  │
│   │                                                                                      │  │
│   │   特点: 高吞吐要求、性能敏感、横向扩展、执行角色                                       │  │
│   └─────────────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                              │
│   设计原则:                                                                                   │
│   • 控制平面故障不应影响已有数据平面流量                                                       │
│   • 数据平面可以独立于控制平面运行一段时间                                                     │
│   • 控制平面变更通过渐进式发布降低风险                                                         │
│                                                                                              │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
```

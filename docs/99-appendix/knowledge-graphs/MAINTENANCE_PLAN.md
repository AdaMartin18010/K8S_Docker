```

│                          内容更新流程                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  1. 发现变更                                                                 │
│  ════════════════════════════════════════════════════════════════════════   │

│  │      ▼                                                                     │
│  │ 创建更新任务 (Issue)                                                        │

│  │                                                                            │
│  ════════════════════════════════════════════════════════════════════════   │
│                              │                                               │
│  2. 影响评估                                                 │               │
│  ════════════════════════════════════════════════════════════════════════   │

│  │   - 哪些文档需要更新?                                                       │
│  │   - 是否需要新增内容?                                                       │
│  │   - 是否有废弃内容需要标记?                                                  │
│  │   - 示例代码是否受影响?                                                      │
│  │                                                                            │
│  │ 输出: 影响分析文档                                                           │
│  │                                                                            │
│  ════════════════════════════════════════════════════════════════════════   │
│                              │                                               │
│  3. 内容更新                                                 │               │
│  ════════════════════════════════════════════════════════════════════════   │
│  │ 按照模板更新内容:                                                           │

│  ════════════════════════════════════════════════════════════════════════   │
```

## 👥 贡献者指南

### 如何贡献

```
贡献类型:
├── 内容更新
│   ├── 修复错误 (错别字、错误命令)
│   ├── 更新版本 (新版本特性)
│   └── 补充示例 (更多使用场景)
│
├── 新增内容
│   ├── 新工具/技术介绍
│   ├── 最佳实践案例
│   └── 故障排查指南
│
└── 结构优化
    ├── 改进导航
    ├── 优化图表
    └── 翻译工作
```

### 贡献流程

1. Fork 仓库
2. 创建分支 (`feature/update-k8s-1.34`)
3. 提交变更 (遵循 commit 规范)
4. 创建 Pull Request
5. 代码审查
6. 合并发布
